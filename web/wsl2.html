🐧 WSL 2 erklärt

WSL 2 steht für Windows Subsystem for Linux, Version 2. Es ist eine von Microsoft entwickelte Technologie, die es ermöglicht, einen vollständigen Linux-Kernel direkt unter Windows 10/11 auszuführen — ohne Dual-Boot, ohne separate Partition, ohne Neustart.

Unter der Haube läuft ein echter Linux-Kernel in einer leichtgewichtigen, verwalteten virtuellen Maschine. Für den Nutzer fühlt es sich wie ein normales Terminal-Fenster an.

⚡ Warum WSL 2?

Viele Entwickler-Tools (Git, Python, Node, Docker) laufen unter Linux stabiler und schneller als unter Windows. WSL 2 gibt Entwicklern das Beste aus beiden Welten.

🐚 Was läuft darin?

Standardmäßig Ubuntu — mit vollem Zugriff auf apt, Bash, Python, Git und alle Linux-Kommandozeilen-Tools. Andere Distributionen (Debian, Kali, openSUSE) sind ebenfalls verfügbar.

🔧 Installation

Ein einziger Befehl in PowerShell (als Administrator) reicht:

wsl --install

🖥️ Unterschied zu einer virtuellen Maschine

Obwohl WSL 2 intern einen Linux-Kernel in einer VM betreibt, unterscheidet es sich fundamental von einer klassischen VM wie VirtualBox oder VMware.

Merkmal Klassische VM WSL 2
Startzeit 1–3 Minuten 1–2 Sekunden
Ressourcenverbrauch Feste RAM/CPU-Zuweisung Dynamisch, teilt sich mit Windows
Grafische Oberfläche Vollständiger Linux-Desktop Terminal (+ WSLg für GUI-Apps)
Dateizugriff auf Windows Shared Folders, langsam Direkt über /mnt/c/
Integration in Windows-Tools Keine (isoliert) Tief — VS Code, Terminal, Explorer
Netzwerk Eigenes virtuelles Netzwerk Teilt sich mit Windows
Konfigurationsaufwand Hoch (ISO, Disk, RAM) Minimal (wsl --install)
Kurzfassung: Eine klassische VM ist ein vollständig isolierter Computer im Computer. WSL 2 ist eine tief in Windows integrierte Linux-Umgebung — schneller, leichter und einfacher zu bedienen.

🌍 Die zwei Welten: Windows und Linux

Mit WSL 2 laufen zwei Betriebssysteme gleichzeitig. Beide haben ihr eigenes Dateisystem, ihre eigenen Programme und ihre eigene Paketverwaltung — können aber miteinander kommunizieren.

🪟 Windows-Welt

  • Programme: .exe, installierer
  • Paketverwaltung: winget, Chocolatey
  • Dateisystem: C:\Users\...
  • Tools: VS Code, Explorer, Git für Windows
  • Terminal: PowerShell, cmd

🐧 Linux-Welt (WSL 2)

  • Programme: Linux-Binaries, Shell-Scripts
  • Paketverwaltung: apt, pip, npm
  • Dateisystem: /home/benutzername/
  • Tools: Bash, Git (Linux), Python, Node
  • Terminal: Bash, zsh
Brücke zwischen den Welten: Windows-Programme können aus WSL heraus aufgerufen werden (explorer.exe . öffnet den Explorer im aktuellen Linux-Ordner) und umgekehrt kann Windows auf das Linux-Dateisystem zugreifen.

📁 Dateizugriff über /mnt/c/

WSL 2 hängt das Windows-Dateisystem automatisch unter /mnt/ ein. Die Windows-Laufwerke sind damit direkt aus dem Linux-Terminal erreichbar.

# Windows-Laufwerke in WSL /mnt/c/ → C:\ /mnt/d/ → D:\ # Beispiel: Zum Windows-Desktop navigieren cd /mnt/c/Users/Stephan/Desktop # Beispiel: Projektordner auf C: öffnen cd /mnt/c/Users/Stephan/Documents/knowledge-base # Windows Explorer im aktuellen Linux-Verzeichnis öffnen explorer.exe .

⚡ Empfehlung: Linux-Dateisystem nutzen

Projekte im Linux-Dateisystem (~/projekte/) sind deutlich schneller als im Windows-Dateisystem (/mnt/c/...). Git-Operationen und Build-Tools profitieren besonders davon.

🔗 Zugriff von Windows auf Linux

Im Windows Explorer die Adressleiste öffnen und \\wsl$\Ubuntu\ eingeben — das öffnet das Linux-Dateisystem direkt im Explorer.

Leistungshinweis: Dateien, die intensiv von Linux-Tools bearbeitet werden (Git, npm, Python), sollten im Linux-Dateisystem liegen (/home/...), nicht unter /mnt/c/. Der Zugriff über die Grenze ist langsamer.

💻 WSL 2 Integration in VS Code

VS Code bietet mit der WSL-Extension eine nahtlose Integration: Der Editor läuft unter Windows, aber Dateisystem, Terminal und Git-Befehle laufen vollständig in Linux.

# WSL-Extension in VS Code installieren code --install-extension ms-vscode-remote.remote-wsl # Projektordner in Ubuntu/WSL öffnen — VS Code startet im WSL-Modus cd ~/mein-projekt code .

🟢 Erkennung des WSL-Modus

In der unteren linken Ecke von VS Code erscheint ein grünes Label WSL: Ubuntu — damit ist der WSL-Modus aktiv. Alle Terminals, die jetzt geöffnet werden, sind Bash-Terminals in Linux.

🔌 Extensions im WSL-Modus

Manche Extensions müssen explizit für WSL installiert werden. VS Code zeigt das an — ein Klick auf „In WSL installieren" genügt. GitLens zum Beispiel muss so einmalig nachinstalliert werden.

🖥️ Integriertes Terminal

Das VS Code Terminal (Strg+ö oder Strg+`) öffnet direkt eine Bash-Shell in Linux. Git-Befehle laufen hier in der Linux-Umgebung — verlässlicher als über die Windows-Git-Integration.

Tipp: Einmal code . aus dem WSL-Terminal ausgeführt, öffnet VS Code künftig automatisch im WSL-Modus für diesen Ordner — die Einstellung wird gespeichert.
